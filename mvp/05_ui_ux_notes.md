# 05 – UI/UX-Notizen für das MVP

## Grundprinzipien
- mobile first
- sehr wenige Klicks
- keine Popups als Hauptmechanik
- Warnungen über Inline-Banner, Hinweise, Chips, Statusboxen
- Schichtrelevantes auf einer Seite bündeln

## Mitarbeiterseite – aktuelle Schicht

### Kernaktionen oben sichtbar
- Einstempeln
- Pause starten
- Pause beenden
- Ausstempeln

### Sichtbare Infos
- geplanter Schichtzeitraum
- Bereich
- aktueller Status
- bisherige Arbeitszeit
- bisherige Pause
- Hinweis auf gesetzlich noch nötige Pause
- offene Pflichtpunkte
- offene Aufgaben für diese Schicht

## Schichtbericht-Seite

### Struktur
1. Schichtkopf
2. Pflichtcheckliste
3. Kassen-/Tagesangaben
4. Freitextbericht
5. besondere Vorkommnisse
6. offene Aufgaben
7. optionaler Fotobeleg

### UX-Vorgaben
- fehlende Pflichtangaben farblich/visuell deutlich markieren
- Abschnitte einklappbar, aber Status sichtbar
- keine modalen Dialoge für Standardaktionen

## Admin-Dashboard

### Widgets / Listen
- aktuell eingestempelt
- heute eingeplant
- Schichten unvollständig
- fehlende Pflichtaufgaben heute
- offene Urlaubsanträge
- kritische Zeit-/Pausenwarnungen
- Stundenübersicht Woche

## Aufgabenansicht
- Listenansicht mit Filtern
- klare Trennung „heute zwingend“ vs. „flexibel“
- farbliche Prioritätsmarkierung
- Statuschips statt unnötiger Dialoge

## Mitarbeiterprofil
- Stammdaten
- Rollen
- Vorgesetzter
- Wochenstunden / Sollzeit
- Achievements
- letzte Schichtberichte / Auffälligkeiten
